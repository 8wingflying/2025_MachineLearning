```python
import pandas as pd
import os

DATA_DIR = 'ml-10M100K'
SEPARATOR = '::'

# Read movies.dat
movies_df = pd.read_csv(os.path.join(DATA_DIR, 'movies.dat'),
                        sep=SEPARATOR,
                        engine='python', # 'python' engine is needed for multi-character delimiters
                        names=['MovieID', 'Title', 'Genres'],
                        encoding='latin-1') # Specify encoding if there are special characters

print('Movies DataFrame head:')
display(movies_df.head())
```

```python
# Read ratings.dat
ratings_df = pd.read_csv(os.path.join(DATA_DIR, 'ratings.dat'),
                          sep=SEPARATOR,
                          engine='python',
                          names=['UserID', 'MovieID', 'Rating', 'Timestamp'])

print('\nRatings DataFrame head:')
display(ratings_df.head())
```
```
movies_df (電影資料)
這個 DataFrame 包含了電影的基本資訊，來自 movies.dat 檔案。它有以下幾欄：

MovieID: 電影的唯一識別碼。這是一個整數，用於區分不同的電影。
Title: 電影的標題，通常包含上映年份。
Genres: 電影的類型。一部電影可以有多種類型，這些類型會以 | 符號分隔（例如：Action|Adventure|Sci-Fi）。
這個 DataFrame 的目的是提供電影的元數據，例如它的類別，這在推薦系統中對於了解電影特性非常重要，特別是在冷啟動（cold-start）情境下。

ratings_df (評分資料)
這個 DataFrame 記錄了用戶對電影的評分資訊，來自 ratings.dat 檔案。它有以下幾欄：

UserID: 執行評分動作的用戶的唯一識別碼。這是一個整數，用於區分不同的用戶。
MovieID: 被評分的電影的唯一識別碼。這個 ID 與 movies_df 中的 MovieID 相對應。
Rating: 用戶給予電影的評分。通常是一個浮點數，例如 0.5 到 5.0 之間。
Timestamp: 評分行為發生的時間戳，以 Unix 時間表示。這表示了評分的時間點。
這個 DataFrame 是推薦系統的核心，它包含了用戶與電影之間的互動數據，是建立協同過濾（collaborative filtering）模型和學習用戶偏好的基礎。
```

## 探索性資料分析（EDA）
- 這個分析將包括資料概覽、評分分佈、電影類型分析、電影和用戶活躍度分析，以及評分的時間趨勢。
- 資料概覽: 檢查 movies_df 和 ratings_df 的基本資訊，包括資料類型、缺失值、描述性統計，以及關鍵欄位的唯一值數量，以初步了解資料結構和品質。
- 評分分佈分析: 分析 ratings_df 中 Rating 欄位的分佈情況，包括繪製評分值的直方圖，以了解用戶給予評分的一般趨勢。
- 電影類型分析: 處理 movies_df 中的 Genres 欄位，將多個類型拆分為獨立的項目，然後統計各類型電影的出現頻率，並繪製最常見電影類型的長條圖。
- 電影和用戶活躍度分析: 分析電影被評分的數量和用戶的評分次數。這包括找出評分次數最多的電影、平均評分最高的電影（考慮最低評分次數以避免偏頗），以及最活躍的用戶（評分次數最多的用戶），並繪製相關的長條圖以視覺化結果。
- 評分時間趨勢分析: 將 ratings_df 中的 Timestamp 欄位轉換為日期時間格式，然後分析隨著時間推移的評分趨勢，例如每年或每月評分數量的變化，並繪製折線圖。
- Final Task: 根據上述 EDA 分析結果，提供關鍵發現和洞察，幫助進一步理解資料集的特性。
